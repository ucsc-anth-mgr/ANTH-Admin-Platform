Import legacy service data

Import the legacy service database CSV (headers: First Name, Last Name, Service Category, Role, Year, Quarter). Names are matched against portal profiles, including alternate spellings; unmatched names are kept as historical records. Unknown categories are created automatically; non-quarter values in the Quarter column are moved into Notes. The import appends — exact duplicates are skipped.

Import legacy self-nominations

Import the legacy nominations CSV (headers: Year, First Name, Last Name, Service Category, Role, Quarter, Notes). Rows land as OPEN nominations for the year in the CSV — a live queue to accept (which creates that year's assignments) or decline. The legacy form was checkbox-style, so priorities follow each person's row order. Nominations require a portal profile: unmatched names are skipped — use Link in the preview to bind a spelling first. Duplicates (same person, category, and year) are skipped.
